# Wiki CMS

**CMS Wiki** — простая CMS для создания wiki-сайтов, справочников, баз знаний и SEO-контентных проектов на Laravel.

Система позволяет создавать и редактировать страницы, связывать их между собой внутренними ссылками, использовать визуальный редактор, загружать изображения и контролировать доступность страниц для поисковых систем.

---

## Возможности

* создание и редактирование wiki-страниц;
* WYSIWYG-редактор для визуального редактирования текста;
* режим ручного редактирования HTML-кода;
* загрузка изображений через редактор;
* сохранение изображений в `public/uploads`;
* поддержка заголовков, списков, цитат, таблиц, ссылок и изображений;
* вывод HTML-контента на публичной странице;
* контроль доступности страниц через внутренние ссылки;
* отдача `404` для страниц, на которые нет прямой ссылки;
* удобная структура для SEO-wiki и справочных сайтов;
* возможность использовать CMS как основу для разных сайтов-дистрибутивов.

---

## Как работает CMS Wiki

В основе CMS лежит простая wiki-логика: каждая страница является отдельной статьёй, которую можно редактировать из административной панели.

Администратор добавляет текст, ссылки, изображения, цитаты и HTML-разметку. Страницы связываются между собой внутренними ссылками, благодаря чему формируется структура сайта.

Если на страницу нет прямой ссылки из других материалов, такая страница может отдавать `404`. Это помогает не допускать появления случайных, пустых или мусорных URL.

---

## WYSIWYG-редактор

В админке используется визуальный редактор для удобной работы с контентом.

Редактор позволяет:

* форматировать текст;
* добавлять заголовки;
* вставлять ссылки;
* создавать списки;
* оформлять цитаты;
* добавлять таблицы;
* загружать изображения;
* переключаться в режим HTML-кода.

При необходимости можно вручную отредактировать HTML-разметку: например, поправить `<blockquote>`, `<img>`, `<a>`, `<h2>`, `<p>` и другие теги.

---

## Загрузка изображений

Изображения, вставленные через редактор, загружаются в директорию:

```text
public/uploads
```

После загрузки изображение автоматически вставляется в текст статьи в виде HTML:

```html
<img src="/uploads/example.jpg" alt="">
```

На публичной странице изображения отображаются внутри контента и адаптируются под ширину блока.

---

## Контроль страниц и 404

CMS может проверять, есть ли на страницу прямая ссылка в текстах других страниц.

Если ссылка есть — страница доступна.

Если ссылки нет — страница отдаёт:

```http
404 Not Found
```

Это полезно для SEO, потому что помогает:

* не индексировать случайные URL;
* не плодить пустые страницы;
* контролировать структуру сайта;
* улучшать качество внутренней перелинковки;
* оставлять в индексе только нужные страницы.

## Преимущества

### Простая структура

CMS не перегружена лишними сущностями. В основе — страницы, контент и внутренние ссылки.

### Удобное редактирование

Администратор может работать с текстом в визуальном режиме, не редактируя HTML вручную.

### HTML-режим

Для точной настройки разметки доступен режим HTML-кода.

### Поддержка изображений

Картинки можно загружать прямо из редактора. Они сохраняются в `public/uploads` и вставляются в статью автоматически.

### SEO-контроль

Страницы без прямых внутренних ссылок могут отдавать `404`, что помогает избежать мусорной индексации.

### Подходит для дистрибутивов

CMS можно использовать как основу для разных сайтов: SEO-wiki, справочников, каталогов, баз знаний и тематических информационных проектов.

## Пример вывода HTML на фронте

Чтобы сохранённые HTML-теги отображались корректно, контент нужно выводить без экранирования:

```blade
{!! html_entity_decode($text) !!}
```

Если использовать обычный вывод:

```blade
{{ $text }}
```

HTML-теги будут отображаться как простой текст.

## Структура загрузок

Папка для изображений:

```text
public/uploads
```

Рекомендуется убедиться, что у папки есть права на запись:

```bash
chmod -R 755 public/uploads
```

Если сервер не даёт Laravel сохранять файлы, можно использовать:

```bash
chmod -R 775 public/uploads
```

## Перенос на другие сайты

Редактор и базовую логику CMS можно использовать на других Laravel-сайтах.

Для переноса обычно нужны:

* Blade-шаблон формы редактирования;
* JS-код инициализации редактора;
* маршрут загрузки изображений;
* метод загрузки изображений в контроллере;
* вывод HTML на фронте;
* папка `public/uploads`.

Если другой сайт тоже использует Laravel и Blade, перенос выполняется с минимальной адаптацией.
